## Goal

Operator can see "SQLite has X new activities, Y changed, Z removed" vs prod PG in the admin UI, with drill-down per lesson/part/day/activity. No writes yet.

## Status

**Code:** complete. **Test:** run [P0A_Manual_Testing.md](P0A_Manual_Testing.md) then [P0_Manual_Testing.md](P0_Manual_Testing.md).

## Steps

### 0. Local dev baseline

- [x] PostgreSQL 16 on `localhost:5432` (operator provides).
- [x] Redis 7 on `localhost:6379` (operator provides).
- [x] `yyddp-api` runs via `pnpm dev` on `:8888`.

- [ ] Dev smoke green end-to-end — see [P0A_Manual_Testing.md](P0A_Manual_Testing.md). **← user runs this**

### 0.5 Dev-loop script: fetch latest SQLite

- [x] New: `yyddp-importer/scripts/fetchLatestSqlite.ts`
- [x] Script command: `pnpm run fetch:latest-sqlite`
- [x] Behavior:
  - [x] Lists `oss://ailanguages/videos/sqlite/` (prefix from `$SQLITE_OSS_PREFIX`, default `videos/sqlite/`).
  - [x] Picks the highest-dated `YYYYMMDD.sqlite`, skipping `input.sqlite` (dev cruft).
  - [x] Downloads to `$SQLITE_INPUT_PATH` (default `./input.sqlite`).
  - [x] Prints source file + size + timestamp.
- [x] Logic shared in `src/oss/fetchSqlite.ts` so the P5 scheduled worker can reuse it.

### 1. PG-side JSON generator

- [x] New: `yyddp-importer/src/exporters/exportFromPg.ts` — reads courses, lessons, parts, days, activities, interactions via Prisma.
- [x] Output shape matches `data.json` (Activity[]) so diff is a straight compare.
- [x] Prisma schema: copied from `yyddp-api/prisma/schema.prisma` to `yyddp-importer/prisma/schema.prisma`.
- [ ] Prisma migrations applied to dev PG: `cd yyddp-api && pnpm prisma:migrate:deploy`. (Deferred to P1 — exporter handles empty PG gracefully.)
- [x] Script: `pnpm run export:pg` → `pg-data.json`.

### 2. Diff engine

- [x] New: `yyddp-importer/src/diff/diffData.ts` — deep-diff `data.json` vs `pg-data.json`, keyed by activity URL.
- [x] Classify: `added` / `removed` / `changed` (per-field) / `unchanged`.
- [x] Output: `diff.json`.
- [x] Script: `pnpm run diff`.
- [x] Deterministic — same input = byte-identical output (canonical key sort).

### 3. Admin UI diff view

- [x] `yyddp-importer/server/index.ts` — `GET /api/pg-data`, `GET /api/diff`.
- [x] New Vue components: `DiffView.vue`, `DiffBadge.vue`.
- [x] Diff tab beside Preview:
  - [x] Left: nav tree with add/remove/change badges on Lesson / Part / Day / Activity rows.
  - [x] Right: side-by-side JSON with changed-field list.
- [x] No Import button — that's P1.

## Risks

- [ ] **Schema drift** — the importer's `prisma/schema.prisma` is a hand-copy. When the API schema changes, re-copy and re-run `pnpm run prisma:generate`.
- [ ] **Prod PG unreachable during dev** — `pg-data.json` comes from local dev PG; same code path, different `DATABASE_URL` in prod.

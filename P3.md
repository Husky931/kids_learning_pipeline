## Goal

Every endpoint in Contract §2.3–§2.6 has a failing integration test against a real Postgres fixture. Tests define acceptance criteria for P4 and P5.

## Prerequisites

- [ ] P1 complete, so a seed script can reuse the importer's loader instead of hand-crafting SQL fixtures.

## Steps

### 1. Test strategy reset

- [x] Keep Mocha + Chai + chai-http (no Jest migration — waste of a day).
- [x] `origin/dev` already has skeletal tests in `test/services/authService.test.js`, `test/auth.test.js`, `test/user.test.js`, `test/courses.test.js`. Expand, don't delete. Most mock the service layer; convert to hit `test-pg`.
- [ ] For every endpoint:
  - [ ] (a) auth required → 401 when no JWT
  - [ ] (b) happy path returns PG row
  - [ ] (c) 404/403 where applicable
  - [ ] (d) idempotency where applicable (P5)
- [ ] Use supertest + the real Express app + real Prisma client pointing at the `test-pg` container.
- [ ] No service-layer mocks. Delete `yyddp-api/src/db/mockups/` fixtures as they're replaced.

### 2. Test harness

- [ ] New: `yyddp-api/test/helpers/db.ts` — wraps each test in a Prisma transaction that rolls back on teardown. Fast + isolated.
- [ ] New: `yyddp-api/test/helpers/auth.ts` — factory `mintJwt(userId)` that returns a valid JWT without going through SMS.
- [ ] New: `yyddp-api/prisma/seed.ts` — deterministic: 2 courses, 3 users, 1 locked activity, 1 unlocked.

### 3. Test database runner

- [ ] Dedicated `yyddp_test` database on the local Postgres. Create once: `createdb yyddp_test`.
- [ ] `TEST_DATABASE_URL=postgresql://yyddp:devpassword@localhost:5432/yyddp_test` in `.env.dev`.
- [ ] Runner: `pnpm test` — drops+recreates `yyddp_test` (via `prisma migrate reset --force --skip-seed`), re-migrates, seeds, runs Mocha.
- [ ] CI hook: same command (`.travis.yml` already runs `npm test` on Node 20; update it to provision `yyddp_test` first).

## Definition of Done

- [ ] `pnpm test` passes locally.
- [ ] Red tests exist for every endpoint P4 and P5 must implement.
- [ ] Test isolation: one failing test doesn't leak state into the next.

## Risks

- [ ] **Scope creep to Jest migration** — explicitly rejected above.
- [ ] **Seed-vs-loader divergence** — mitigated by reusing P1's loader in `seed.ts` where possible; otherwise keep seed minimal.

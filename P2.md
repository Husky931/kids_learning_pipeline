## Goal

The mobile app can record learning activity — start a session, submit interactions, complete activities, and read back progress. Gold coins are awarded per product rules. No unlock enforcement yet (all content stays unlocked); that lands in P5.

## Steps

### 1. Schema additions

- [x] New table `gold_rules` — maps (interaction_type, activity_type, first_time_vs_replay) → coin amount. Seeded with defaults: 1 coin per correct interaction, 10 per activity completion.
- [x] New table `user_gold_ledger` — append-only: (user_id, source_type, source_id, amount, created_at). Unique index on `(user_id, source_id)` for replay protection.
- [x] Run `prisma db push` to apply (used db push instead of migrate dev since schema was pulled from existing DB).

### 2. Gold service

- [x] New: `src/session/services/goldService.js`.
- [x] Single entry point: `award(client, userId, sourceType, sourceId, amount)` — accepts a Prisma client or transaction.
- [x] Writes one row to `user_gold_ledger`.
- [x] Atomically increments `users.coin`.
- [x] Replay-safe: if `(user_id, source_id)` already exists, return success without awarding again.

### 3. Session service + routes

- [x] New: `src/session/services/sessionService.js`.
- [x] New: `src/api/v1/sessions/index.js` (routes) + `src/session/controllers/sessionController.js`.

#### `POST /api/v1/sessions/start`
- Creates a `user_activity_learning_sessions` row.
- Body: `{ activity_id, day_id }`.
- Returns: `{ learning_session_id }`.
- Validates activity exists and belongs to user's current course.

#### `POST /api/v1/sessions/{id}/interactions`
- Records one interaction result.
- Body varies by type:
  - **video**: `{ interaction_id, video_watched_seconds, watched_percentage }`
  - **audiopickcard**: `{ interaction_id, selected_card_id, is_correct, response_time }`
  - **audiopickimage**: `{ interaction_id, selected_image_id, is_correct, response_time }`
  - **wordcombo**: `{ interaction_id, attempt_count, correct_count, incorrect_count }`
- Looks up `gold_rules` for per-interaction coin award; calls `goldService.award()`.
- Returns: `{ recorded: true, coins_awarded: N }`.

#### `POST /api/v1/sessions/{id}/complete`
- Marks the learning session as completed.
- Sets `completed_at`, `time_spent`, `reward_coins` on the session row.
- Updates `user_progress` for this activity to `COMPLETED`.
- Awards activity-completion gold via `goldService.award()`.
- **Idempotent**: if session already completed, returns success without re-awarding.
- Returns: `{ completed: true, total_coins_awarded: N }`.

#### `GET /api/v1/courses/{id}/progress`
- Returns per-activity progress for the user in the given course.
- Joins `user_progress` with course structure.
- Returns: list of `{ activity_id, status, coins_awarded }` grouped by part/day.

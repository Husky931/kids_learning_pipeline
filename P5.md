## Goal

Real SMS OTP login works on a real phone (test target: **+8613162908096**). Unlock engine is live ‚Äî completing an activity unlocks the next one per product rules, with env-configurable end-of-Part timing. Server is fully stateless: all auth/session state in Redis or PG. ### ¬ß3. Auth service ‚Üí Redis-backed

- [x] Add `_normalizePhone()` helper ‚Äî strips leading `+86` before `_assertPhone`. So both `+8613162908096` and `13162908096` land valid. All auth endpoints that accept `phone` normalize on entry.
- [x] `captcha:{id}` ‚Üí `redis.set(value, 'EX', 300)`. Delete on successful verify.
- [x] `otp:{phone}` ‚Üí `redis.set(JSON.stringify({code, attempts}), 'EX', 300)`. Increment attempts on wrong guess; lock after 5.
- [x] `otp-rl:phone:{phone}` ‚Üí `incr` + `expire 60`; reject with 429 if >1 per 60s.
- [x] `otp-rl:ip:{ip}` ‚Üí `incr` + `expire 3600`; reject with 429 if >10/hour per IP.
- [x] Refresh tokens carry a `jti` (uuidv4) in payload. `logout` ‚Üí `redis.set('jti-revoked:{jti}', '1', 'EX', remaining)` with TTL = remaining refresh lifetime. `refresh` ‚Üí `redis.get` check ‚Üí 401 if revoked.
- [x] Also persist active sessions to `user_sessions` PG table on login; flip `is_active=false` on logout (audit survives Redis TTLs).
- [x] **Grep gate** ‚Äî post-migration, zero hits in `src/` for: `this.captchaData`, `this.otpData`, `this.tokenData`, `captchaData: asValue`, `otpData: asValue`. Server is stateless.

### ¬ß6. Unlock engine

**Source of truth:** `user_progress` (existing). Day/Part/Course unlock state is **derived** from it, not stored separately. `unlock_logs` remains the append-only audit log.

- [x] `prisma/schema.prisma`: add `unlocks_at DateTime?` to `user_progress` (null = immediately available if `status=UNLOCKED`; non-null = unlocks at that timestamp). `prisma db push` to apply.
- [x] New `src/utils/tzHelper.js` ‚Äî `nextBoundary(tz, hour)` returns next `hour:00` in `tz` as a UTC Date, using `Intl.DateTimeFormat`. No moment-timezone dep.
- [x] New `src/services/unlockService.js`:
  - `computeUnlockedState(userId, courseId)` ‚Üí `Map<activityId, {unlocked, unlocks_at}>`, derived from `user_progress` + hierarchy joins.
  - `recomputeAfterCompletion(tx, userId, activityId)` ‚Äî called inside `sessionService.completeSession`'s existing transaction, right after gold award + user_progress upsert.
- [x] Rules:
  - Sequential within Day: activity N+1 becomes UNLOCKED when N is COMPLETED.
  - Next Day unlocks when **all** activities in current Day are COMPLETED (all required ‚Äî no `required` flag in schema, so every activity counts).
  - Next Part's Day 1 unlocks when previous Part fully cleared.
  - End-of-Part / end-of-arc: `unlocks_at = tzHelper.nextBoundary(UNLOCK_TZ, UNLOCK_HOUR)`. Env defaults `Asia/Shanghai` / `10`. **Env-configurable ‚Äî not hardcoded.**
  - Lazy eval, no cron. Every course/part/activity GET runs `computeUnlockedState`; time-delayed rows flip unlocked on the first read after their `unlocks_at`.
- [x] Response-shape changes ‚Äî inject `{ unlocked: bool, unlocks_at: timestamp|null }` at course/part/day/activity granularity in:
  - `src/course/services/courseService.js`
  - `src/parts/services/PartsService.js`
  - `src/activities/services/ActivitiesService.js`
- [x] `UNLOCK_BYPASS=true` stays as the demo escape hatch (short-circuits to all-unlocked). Default `false` in `.env.production` once ¬ß6 lands and tests are green.

## Risks

- [ ] **Aliyun SMS template approval lead time** ‚Äî mitigated, reusing sibling's approved `SMS_504055089`.
- [ ] **Real Aliyun creds required for the live SMS test** ‚Äî user populates `.env.local`. Don't commit.
- [ ] **Unlock engine complexity** ‚Äî densest piece of the sprint. Budget a full day. Mitigated by reusing `user_progress` instead of introducing new tables.
- [ ] **10:00 Beijing assumption** ‚Äî mitigated by making it env-configurable (`UNLOCK_TZ`, `UNLOCK_HOUR`). Client can tune without a code change.
- [ ] **Media upload failures mid-run** ‚Äî idempotent via `media_processing_log` unique index; operator retries `upload:media` without re-running `import:pg`.
- [ ] **OSS prod bucket reachability** ‚Äî P4 never tested actual downloads. Closed in ¬ß7 with `oss-smoke.js`.
- [ ] **Test redis container** ‚Äî CI needs Docker available. `docker-compose.test.yml` pins the image and port.
